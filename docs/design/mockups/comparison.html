VARI vegetation index ramp (the most consequential difference)

Interface-artisanpurple → tan → green
VARIVisible Atmospherically Resistant Index
−1.0−0.50.0+0.5+1.0
Green = healthy (convention preserved). Color-blind safe by luminance. Tan midpoint reads as "neutral / bare soil." GIS-literate users read it on first glance.
voltagent ui-designerbrown → paper → blue
VARIVisible Atmospherically Resistant Index
−1.0−0.50.0+0.5+1.0
Blue = healthy (convention broken). Most color-blind safe of the two. Requires legend education; voltagent proposed an opt-in "classic" red-yellow-green toggle.

What this is. A faithful side-by-side render of the design tokens, fonts, and component patterns from each designer agent. The map canvas is a CSS gradient mockup, not a real MapLibre instance — close enough to feel the design direction.

What's missing. Mobile breakpoints, all 8 screens (only S1 Auth and S3 Map shell rendered), full data tables (S5/S6), the upload wizard wait states (S7). If a direction is picked, the next pass can expand.

Decision points to look at.

  • Serif vs mono wordmark — which one reads as "stewardship" to you?
  • USGS blue (#1E5A8C, muted) vs Tailwind blue (#2563EB, saturated)
  • Green-healthy VARI ramp (intuitive, convention-following) vs blue-healthy (most color-blind safe, unconventional)
  • Hairline-grid texture in empty states (interface-artisan) vs pure negative space (voltagent)
  • Three earth tones with one olive-green (interface-artisan's "soil + lichen") vs two browns (voltagent's "prairie + loam")
